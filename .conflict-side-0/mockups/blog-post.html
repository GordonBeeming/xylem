Building a Custom CLI Tool with Go and Cobra

GB
Gordon Beeming · March 10, 2026 · 8 min read
go cli developer-tools cobra tutorial

Building command-line tools is one of the most rewarding types of software development. There's something satisfying about creating a tool that does exactly what you need, runs instantly, and can be composed with other tools in your workflow. In this post, I'll walk through building a production-ready CLI tool using Go and the Cobra library.

Why Go for CLI Tools?

Go has become the go-to language for building command-line tools, and for good reason. The language compiles to a single static binary, which means your users don't need to install a runtime or manage dependencies. Compilation is fast, cross-compilation is built in, and the standard library provides everything you need for file I/O, networking, and text processing out of the box.

Go's ability to compile to a single static binary makes it the perfect choice for distributing CLI tools. No runtime dependencies, no installation headaches.

Setting Up the Project

Cobra provides a clean, idiomatic way to structure your CLI application. Each command is self-contained, with its own flags, arguments, and help text. Let's start with the standard project layout and a root command that serves as the entry point for our tool.

main.go go 
 1package main
 2
 3import (
 4    "fmt"
 5    "os"
 6
 7    "github.com/spf13/cobra"
 8)
 9
10var rootCmd = &cobra.Command{
11    Use:   "mytool",
12    Short: "A brief description of your tool",
13    Long:  `A longer description that spans multiple lines
14and provides more detailed information about your CLI tool.`,
15    Run: func(cmd *cobra.Command, args []string) {
16        fmt.Println("Hello from mytool!")
17    },
18}
19
20func main() {
21    if err := rootCmd.Execute(); err != nil {
22        fmt.Fprintln(os.Stderr, err)
23        os.Exit(1)
24    }
25}

The root command is the entry point of your CLI. When a user runs mytool without any subcommands, the Run function executes. The Use, Short, and Long fields control the help output that Cobra automatically generates.

Adding Subcommands

The real power of Cobra comes from its subcommand architecture. Each subcommand lives in its own file, has its own flags, and can define argument validation rules. This pattern keeps your codebase organized as your CLI grows. Think of commands like git commit, git push, and git log — each is a subcommand with distinct behavior.

cmd/greet.go go 
 1package cmd
 2
 3import (
 4    "fmt"
 5    "strings"
 6
 7    "github.com/spf13/cobra"
 8)
 9
10var greetCmd = &cobra.Command{
11    Use:   "greet [name]",
12    Short: "Greet someone by name",
13    Args:  cobra.ExactArgs(1),
14    Run: func(cmd *cobra.Command, args []string) {
15        uppercase, _ := cmd.Flags().GetBool("uppercase")
16        name := args[0]
17        greeting := fmt.Sprintf("Hello, %s! 👋", name)
18        if uppercase {
19            greeting = strings.ToUpper(greeting)
20        }
21        fmt.Println(greeting)
22    },
23}
24
25func init() {
26    greetCmd.Flags().BoolP("uppercase", "u", false, "Print greeting in uppercase")
27    rootCmd.AddCommand(greetCmd)
28}

Testing Your CLI

Testing is critical for CLI tools, especially when they interact with the file system or external services. Go's built-in testing framework works well with Cobra commands. You can test individual commands by executing them programmatically and capturing their output. Start by running go test ./... to execute your entire test suite.

Terminal bash 
1$ go build -o mytool .
2$ ./mytool greet Gordon
3Hello, Gordon! 👋
4$ ./mytool greet Gordon --uppercase
5HELLO, GORDON! 👋
Tip

You can use goreleaser to automate cross-compilation and release publishing for your CLI tool.

Distribution and Cross-Compilation

One of Go's killer features for CLI development is effortless cross-compilation. A single go build command with the right environment variables produces binaries for any supported platform. Combined with GoReleaser, you can automate the entire release pipeline, including checksums, changelogs, and publishing to package managers.

  • Linux (amd64, arm64)
  • macOS (amd64, arm64)
  • Windows (amd64)

With just a few lines of configuration, GoReleaser handles building for all platforms, creating GitHub releases, and even publishing Homebrew formulae and Scoop manifests. If you're serious about distributing your CLI tool, I highly recommend checking out the GoReleaser documentation to get started.
