# xylem-x

Gordon Beeming's developer blog and portfolio at [gordonbeeming.com](https://gordonbeeming.com).

Built with [Next.js](https://nextjs.org), [TinaCMS](https://tina.io), and [Tailwind CSS](https://tailwindcss.com).

## Prerequisites

- [Node.js](https://nodejs.org/) 22+
- [pnpm](https://pnpm.io/) 10+

## Getting Started

```bash
# Install dependencies
pnpm install

# Start development server (images are copied automatically)
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000).

### With TinaCMS Admin

To enable the CMS admin panel at `/admin` with visual editing:

```bash
pnpm dev:tina
```

This starts the TinaCMS local GraphQL server alongside Next.js. Content edits in the admin write directly to local MDX/JSON files.

## Content

Blog posts are MDX files in `content/blog/YYYY-MM-DD/slug.mdx` with frontmatter:

```yaml
---
title: Post Title
date: 2026-03-11
tags: [tag1, tag2]
summary: Brief description
draft: false
---
```

Code blocks support titles via the meta string:

````markdown
```csharp title="Program.cs"
Console.WriteLine("Hello");
```
````

## Environment Variables

Copy `.env.example` to `.env.local` and configure:

```bash
cp .env.example .env.local
```

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_TINA_CLIENT_ID` | For Tina Cloud | TinaCMS client ID |
| `TINA_TOKEN` | For Tina Cloud | TinaCMS API token |
| `NEXT_PUBLIC_GISCUS_REPO` | For comments | GitHub repo for Giscus |
| `NEXT_PUBLIC_GISCUS_REPOSITORY_ID` | For comments | Giscus repo ID |
| `NEXT_PUBLIC_GISCUS_CATEGORY` | For comments | Giscus category name |
| `NEXT_PUBLIC_GISCUS_CATEGORY_ID` | For comments | Giscus category ID |
| `GITHUB_TOKEN` | Optional | GitHub API token (higher rate limit for project stars) |

## Deployment

### How It Works

Content lives in Git as MDX/JSON files. The build pipeline handles everything automatically:

1. `node scripts/copy-images.mjs` — copies blog images from `content/blog/*/images/` to `public/images/`
2. `next build` — builds the site (static generation for all 184+ posts, feeds, sitemap)

Both steps run automatically via `pnpm build` — no manual image copy needed in CI.

### With TinaCMS Cloud (Production Editing)

For browser-based editing at `/admin` in production:

1. Create a project at [app.tina.io](https://app.tina.io)
2. Connect your GitHub repo
3. Set env vars: `NEXT_PUBLIC_TINA_CLIENT_ID`, `TINA_TOKEN`
4. Use `pnpm build:tina` (runs `tinacms build` before `next build`)

When editors save changes via the TinaCMS admin, Tina Cloud commits to your Git repo, which triggers your CI/CD pipeline to rebuild and deploy.

### Without TinaCMS Cloud

Use `pnpm build` (skips `tinacms build`). Content is edited directly in Git. The `/admin` route won't work in production, but the site renders perfectly from the MDX/JSON files.

### Docker

```bash
docker build -t xylem-x .
docker run -p 3000:3000 xylem-x
```

The Dockerfile runs the full build pipeline (image copy + Next.js build) and produces a standalone Node.js server.

## Tech Stack

- **Framework**: [Next.js](https://nextjs.org) (App Router)
- **CMS**: [TinaCMS](https://tina.io) (Git-backed, visual editing)
- **Styling**: [Tailwind CSS](https://tailwindcss.com) v4
- **Content**: MDX with [Shiki](https://shiki.style) syntax highlighting
- **Analytics**: Google Analytics 4 + Web Vitals
- **Comments**: [Giscus](https://giscus.app) (GitHub Discussions)
- **Fonts**: Inter + JetBrains Mono via `next/font`
- **Deployment**: Docker (standalone output)
