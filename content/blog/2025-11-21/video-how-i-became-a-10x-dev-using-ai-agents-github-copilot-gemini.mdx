## Key concepts

If you don't have time to watch the full session, here are the core ideas I cover:

The goal is to fail fast and ship faster by managing agents rather than writing boilerplate yourself. That means the actual coding is often the last thing you touch.

Gemini's context window is huge, and I use that deliberately. Instead of crafting a precise prompt, I brain-dump my entire messy thought process and let the model clean it up and push back on my assumptions. It's a genuinely different way of working.

One side effect: the "documentation tax" mostly disappears. ADRs and specs come out of your brainstorming sessions almost automatically when you're already having that conversation with an AI.

## The Tooling Ecosystem

Don't use one tool for everything. I break down my workflow based on the risk and creativity required:

* **IDE (Visual Studio / VS Code):** Use for **Deterministic** tasks. Renaming, refactoring, and moving files should be done here because it's guaranteed to be correct.
* **Gemini:** Use for **Creative** tasks and **Planning**. Best for generating new services from scratch or high-level architecture discussions.
* **GitHub Spark:** Use for **Prototyping**. Great for spinning up React apps (like my wife's portfolio) to get a visual base quickly.
* **Tina CMS:** Use for **Content**. Decouples content management from the code so you can use a visual editor while keeping data in Git.

## The "Fix": Securing the Copilot CLI

The GitHub Copilot CLI is powerful but risky. Giving an AI agent full read/write access to your file system and terminal can lead to it nuking files or hallucinating commands.

To use the CLI safely and efficiently, I built a **Secure Portable Copilot CLI Environment**.

<Figure key="/images/docker-copilot-cli-sandbox.png" src="/images/docker-copilot-cli-sandbox.png" alt="Terminal showing the Docker container startup for the Copilot CLI sandbox." width="0" height="0" caption="My secure Docker sandbox for the Copilot CLI" />

### How it works:
1.  **Dockerized Sandbox:** The CLI runs inside a container, not on my host machine.
2.  **Read-Only Mounts:** Critical paths (like `.git` folders or core domain logic) are mounted as read-only. The AI can read them for context but cannot modify them.
3.  **"YOLO Mode":** Because the environment is sandboxed, I can disable the constant "Are you sure?" confirmation prompts. This allows the agent to work in the background autonomously while I focus on other tasks.

## Summary

A modern "10x developer" isn't a lone wolf coding in the dark. It's someone who:
1.  Knows the boundaries of their tools.
2.  Orchestrates agents to do the heavy lifting.
3.  Shares these workflows (and `copilot-instructions.md` files) with their team.
