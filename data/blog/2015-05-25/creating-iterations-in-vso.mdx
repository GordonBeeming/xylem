Click on the gear icon in the top right corner of [VSO](https://www.visualstudio.com/en-us/products/what-is-visual-studio-online-vs)
 
<Figure key="/images/c6c79f6af6104fddac8498a283467c35.png" src="/images/c6c79f6af6104fddac8498a283467c35.png" alt="VSTS dashboard with gear icon highlighted" width="0" height="0" caption="Accessing VSTS settings by clicking the gear icon." /> 
 
Then click on **Iterations**
 
<Figure key="/images/632198f2050e4c0880029bdd09a807d7.png" src="/images/632198f2050e4c0880029bdd09a807d7.png" alt="VSTS settings menu with Iterations option highlighted" width="0" height="0" caption="Selecting Iterations from the VSTS settings menu." /> 
 
You should now see your Iterations for this Team Project
 
<Figure key="/images/2bc7179842f04c2a82a813ed70fe7d59.png" src="/images/2bc7179842f04c2a82a813ed70fe7d59.png" alt="VSTS Iterations page showing existing iterations" width="0" height="0" caption="VSTS Iterations page displaying the current iteration structure." /> 
 
From here you can add New child iterations to an existing iteration
 
 <Figure key="/images/534fdb020f6647e1b7ae6b23b02d9869.png" src="/images/534fdb020f6647e1b7ae6b23b02d9869.png" alt="VSTS Iterations page - New child iteration option" width="0" height="0" caption="Adding a new child iteration in VSTS." /> 
 
<Figure key="/images/736ab1f358c74cdbb8757b870884cfbb.png" src="/images/736ab1f358c74cdbb8757b870884cfbb.png" alt="VSTS Iterations page - New child iteration dialog" width="0" height="0" caption="Entering details for a new child iteration." /> 
 
<Figure key="/images/2faa2b9db43c4df6a1f8855bb78c70fa.png" src="/images/2faa2b9db43c4df6a1f8855bb78c70fa.png" alt="VSTS Iterations page showing newly added child iteration" width="0" height="0" caption="Newly added child iteration in the VSTS iteration structure." /> 
 
Set Iteration dates
 
<Figure key="/images/e0332500553343d291833c290cfaf425.png" src="/images/e0332500553343d291833c290cfaf425.png" alt="VSTS Iterations page - Set dates option for an iteration" width="0" height="0" caption="Setting start and end dates for an iteration." /> 
 
<Figure key="/images/85b2dc6b6b864dd3867a6f3758e39295.png" src="/images/85b2dc6b6b864dd3867a6f3758e39295.png" alt="VSTS Iterations page - Date picker for iteration start and end dates" width="0" height="0" caption="Choosing start and end dates for the iteration." /> 
       
<Figure key="/images/98538bfea96b4e5a912c15eace6b6780.png" src="/images/98538bfea96b4e5a912c15eace6b6780.png" alt="VSTS Iterations page with updated iteration dates" width="0" height="0" caption="Iteration dates updated and displayed in VSTS." /> 
 
Move sprints around with drag and drop
 
<Figure key="/images/9d2b1b7f731f460daf4bd75d97918604.png" src="/images/9d2b1b7f731f460daf4bd75d97918604.png" alt="VSTS Iterations page - Drag and drop functionality for reordering iterations" width="0" height="0" caption="Reordering iterations using drag and drop in VSTS." /> 
 
and select which Iterations are currently visible for this team (not Team Project)
 
<Figure key="/images/3b397253f5dd4f8f97afb7b38ac9e5ce.png" src="/images/3b397253f5dd4f8f97afb7b38ac9e5ce.png" alt="VSTS Iterations page - Selecting visible iterations for the team" width="0" height="0" caption="Choosing which iterations are visible for the current team." /> 
 
You can see this updating if you look at the work tab. You can also add new Iterations in line with the currently selected one by clicking on **New**, fill in sprint details as before and click **Save and close**
  
<Figure key="/images/af8740782cb24ddb85c820c258951abf.png" src="/images/af8740782cb24ddb85c820c258951abf.png" alt="VSTS Iterations page - Adding a new iteration in line" width="0" height="0" caption="Adding a new iteration at the same level as the selected one." /> 
 
All the operations mentioned above are also available from a context menu by right clicking on any iteration or clicking the little arrow to the left of the Iteration
 
<Figure key="/images/a29132ac665c4f0f9a25a81e00ebc380.png" src="/images/a29132ac665c4f0f9a25a81e00ebc380.png" alt="VSTS Iterations page - Context menu for an iteration" width="0" height="0" caption="Context menu options for managing iterations in VSTS." /> 
 
And as you can see in the context menu you can set the team's backlog iteration, just right click on a iteration and choose **Set as team's backlog iteration**
 
<Figure key="/images/d0bce02929da43dfbdef92031e24c9ee.png" src="/images/d0bce02929da43dfbdef92031e24c9ee.png" alt="VSTS Iterations page - Set as team's backlog iteration option" width="0" height="0" caption="Setting an iteration as the team's backlog iteration." /> 
 
Notice how when you do this all iterations outside the tree of the backlog iteration are "unchecked" and not available for inclusion into the teams backlog iterations
 
<Figure key="/images/c1db15b136254a2dac6994bbf906b74a.png" src="/images/c1db15b136254a2dac6994bbf906b74a.png" alt="VSTS Iterations page showing effect of setting backlog iteration" width="0" height="0" caption="Iterations outside the backlog iteration's tree are deselected." /> 
 
This will reflect in the work tab as well because only checked iterations are shown
