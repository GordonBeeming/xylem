## The "Hack" Solution

After checking for hidden `cd` commands in my `.zshrc` and toggling every "Advanced Configuration" option known to man, I went for the brute force approach. ğŸ› ï¸

If iTerm2 won't *start* in the right directory using the standard UI toggle, we'll just tell it to *move* there the split second it opens. 

### The Fix

1. Open iTerm2 **Settings** (`Cmd + ,`).
2. Go to **Profiles** > **General**.
3. Locate the **Send text at start** field.
4. Enter the following:

```zsh
cd /Users/gordonbeeming/Developer/github && clear

```
